# zlm-tinychat-48k

A 48K Spectrum port of `examples/zlm-tinychat`. Same model, same weights —
rearranged so it fits in 41 KB of usable RAM on a stock 48K, with no
visible side-effect on the display.

## Credit

The model itself is HarryR's [Z80-μLM](https://github.com/HarryR/z80ai)
— a 2-bit quantized conversational LM small enough to run on a Z80, with
quantization-aware training, a trigram-hash input encoder, and an
original ZX Spectrum target. Architecture, weights, training pipeline,
and the idea that this should run on a Spectrum at all: all HarryR.
Sincere thanks and warm regards.

The zt-side contribution is the Forth reimplementation and the
memory-map arrangement below.

## Memory map

```
$5B00..$5C00 (256 B)  acts3        — printer-buffer area, invisible
$5C00..$F9E0          image (code + W1..W4 + biases + charset
                              + small buffers + variables)
$F9E0..$FB60 (384 B)  acts2
$FB60..$FD60 (512 B)  acts0
$FD60..$FF60 (512 B)  acts1
$FF80                 return-stack top
$FFC0                 data-stack top
```

The display reads pixels from `$4000..$57FF` and attrs from `$5800..$5AFF`.
None of those addresses get touched by the model, so the screen stays
clean during a forward pass.

## What changed vs. the 128K reference

1. **8-bit signed biases** instead of 16-bit cells (`c@s` sign-extends on
   load). Saves 616 B. Generated by `gen_model.py`.
2. **Big activations hoisted** to fixed addresses outside the image:
   constants like `$FB60 constant acts0` instead of
   `create acts0 512 allot`. Saves ~1.4 KB.
3. **Trimmed stdlib**: `--no-stdlib` plus an inlined `cls` and `cr` only.
   Saves ~200 B by dropping unused `/`, `/mod`, `mod`, `(u.)`, `u.`, `.`,
   `space`, `spaces`.
4. **`--no-inline-next`**: keeps NEXT in one place instead of splicing
   it into every primitive. Saves 747 B.
5. **`acts3` parked at `$5B00`** in the printer-buffer area — plain RAM
   that the display never reads. This is what closes the budget without
   a visible side-effect.

No retraining. No model surgery. Same `model.npz` pipeline.

## Build

```
zt build examples/zlm-tinychat-48k/main.fs -o tc48.sna --target 48k \
    --origin 0x5C00 --rstack 0xFF80 --dstack 0xFFC0 \
    --no-inline-next --no-stdlib
```

Output is a standard 49179-byte `.sna` snapshot loadable by Fuse,
ZEsarUX, or a real 48K via divMMC.

## Tests

`tests/test_examples_zlm_tinychat_48k.py` has 11 tests:

- 9 layout invariants (compile-time): no bank data emitted, image fits
  below `$F9E0`, every hoisted buffer lies outside `[origin, image_end)`,
  `acts3` sits in invisible RAM, stack regions don't collide.
- 2 runtime tests (marked `slow`): boot the SNA in the simulator, type
  `HELLO\r`, and assert the screen shows `HELLO` followed by `HI`.

```
pytest tests/test_examples_zlm_tinychat_48k.py            # fast
pytest tests/test_examples_zlm_tinychat_48k.py -m slow    # runtime
```
